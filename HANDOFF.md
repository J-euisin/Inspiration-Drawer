## 1. 사용자 프로필 (중요)

- **비개발자**입니다. UX 디자이너이며, 코드를 직접 작성하지 않습니다.
- **바이브 코딩** 방식으로 개발합니다. Antigravity IDE 안의 **Claude Code**에게 프롬프트를 넣어 코드를 작성시킵니다.
- 따라서 **Claude(대화 상대)의 역할**은 다음과 같습니다:
  - 사용자가 원하는 바를 듣고 → **Claude Code에 넣을 프롬프트를 대신 작성해주는 것**
  - 화면 캡처를 보고 문제를 진단하고, 원인을 쉬운 말로 설명하는 것
  - 터미널 명령어(git 등)를 만들어주고, 개념을 비전문가 눈높이로 설명하는 것
- **코드를 직접 작성해주는 게 아니라, 프롬프트를 만들어주는 것이 핵심 역할**입니다.
- 개발 기본 개념(단위 px/pt 차이, 캐시, 빌드 에러 등)도 모를 수 있으므로 친절히 설명이 필요합니다.

### 프롬프트 작성 시 지켜온 규칙
- 한국어로 작성
- 문제 → 원인 → 요청 순으로 구조화
- 모바일/PC 구분이 필요한 경우 반드시 명시
- 항상 마지막에 `"이 부분만 수정하고, 다른 잘 작동하는 기능들은 건드리지 말아줘."` 문구 포함
- 큰 구조 변경 시 `"먼저 계획을 설명하고 내 확인을 받고 진행해줘."` 추가
- design.md 팔레트/라운딩 유지 요청 포함

## 2. 프로젝트 개요

- **서비스명**: 영감의 서랍 (Inspiration Drawer)
- **한 줄 설명**: 떠오르는 단상과 감명 깊은 문장을 나만의 카드로 소장하는 영감 아카이빙 플랫폼
- **기술 스택**: Next.js (App Router) + Tailwind CSS + TypeScript
- **배포**: Vercel (`https://inspiration-drawer.vercel.app/`)
- **저장소**: GitHub `J-euisin/Inspiration-Drawer`
- **로컬 경로**: `/Users/petfambranding/Desktop/Inspiration-Drawer`
- **현재 데이터 저장**: 브라우저 localStorage (서버/DB 없음)

### 핵심 개념: 단상 vs 문장
데이터는 `type` 필드로 두 종류를 구분합니다.
- **단상(dansang)**: 짧은 일기처럼 떠오르는 생각. 출처·저자 **없음**.
- **문장(munjang)**: 책·영화·명언에서 수집한 문장. 출처·저자 **있음**.
- ※ type이 없는 과거 데이터는 모두 '문장'으로 처리하도록 되어 있음.

## 3. 화면 구조

상단 GNB(PC) / 하단 탭(모바일) 3개 메뉴:
1. **오늘의 영감** (홈, `/`)
2. **카드 만들기** (`/create`)
3. **나의 서랍** (`/archive`)

### 오늘의 영감 (홈) — 세로 순서
1. 메인 타이틀: "지금, 서랍에 간직하고 싶은 생각이 있나요?"
   - 폰트: Noto Sans KR / 모바일에서 "싶은" 뒤 줄바꿈
2. 설명 문구: "매일의 작은 생각과 기록이 축적되다 보면, 어느날 반짝이는 영감이 가까이 와있을 거에요."
   - 모바일에서 "보면," 뒤 줄바꿈 / 타이틀과의 간격: PC 20px, 모바일 16px
3. **단상 입력창** (Gemini 검색창 스타일, 알약형)
   - Enter = 즉시 기록 / Shift+Enter = 줄바꿈
   - 한글 IME 조합 버그 수정 완료 (조합 중 Enter 무시)
   - 글자수 제한 없음
   - placeholder: PC "떠오르는 생각을 적어보세요. 아주 사소한 것이라도 좋아요." / 모바일 "아주 사소한 것이라도 좋아요."
4. **단상 목록** (채팅창 스타일)
   - 한 줄 표시, 넘치면 말줄임표
   - 우측에 날짜 (영어 형식)
   - 최신이 위, 최대 4개 노출 후 스크롤
   - 영역 상단에 고정 페이드 마스크 (위치 기반, 글자별 opacity 아님)
   - 옅은 보라 테두리로 감싸짐
   - 스크롤바는 영역 안쪽, 가장자리와 4~6px 간격, 페이드 미적용
   - **단상 0개일 때**: 테두리 대신 양 끝이 흐려지는 그라데이션 직선 1개 (간격은 유지)
   - **단상 1개일 때**: 오늘의 문장이 밀리지 않음 (영역 높이가 단상 1개 크기로 고정)
   - **단상 2개부터**: 오늘의 문장이 아래로 밀림
   - 단상 클릭 → 상세 팝업
5. **오늘의 문장 배너**
   - 가로 배너, 반투명
   - 상단: '오늘의 문장' 라벨 + 장르 태그
   - 중앙: 문장 / 아래: 출처·저자
   - 하단: 왼쪽 "이 문장을 담을 수 있는 시간 OO시간 OO분 남음" (자정 기준 실시간 카운트다운), 오른쪽 '+소장하기' 버튼
   - 문장이 길어져도 위쪽 요소는 고정, 배너만 아래로 확장
6. 하단 문구: "영감 靈感 : 창조적인 일의 계기가 되는 기발한 착상이나 자극."

**배경**: Warm Ivory(#FAF8F5) + 입력창 뒤 은은한 보라 그라데이션 블러 (Gemini 스타일)

### 카드 만들기
- 탭: **'단상 기록하기'(왼쪽, 기본 활성)** / '문장 수집하기'(오른쪽)
- 단상 모드: 생각만 입력 / 문장 모드: 문장 + 출처 + 저자 입력
- 폰트 선택 4종: 고딕(Nanum Gothic), 명조(Gowun Batang), 필기체(Nanum Pen Script), 손글씨(Hi Melody)
- 배경: 배경 색상(프리셋 + 직접 입력) / 이미지 업로드 (5MB 이하, 반투명 적용)
- 우측 카드 미리보기 (placeholder: "여기에 문장이 입력됩니다")
- 저장 버튼: "서랍에 담기 →"
- **이탈 경고**: 작성 중 내용이 있을 때 다른 화면 이동 시 확인 팝업 (앱 내 이동 = 커스텀 팝업 / 브라우저 뒤로가기·창닫기 = beforeunload)

### 나의 서랍
- 필터 탭: 전체 / 문장 / 단상 (각 개수 뱃지 표시)
- 날짜별 그룹핑 (영어 날짜 헤더 + "N장")
- 카드 클릭 → 카드 상세 팝업 (dim 배경, 중앙 정렬)

## 4. 주요 인터랙션 규칙

### 팝업 (모달)
| 팝업 | 구성 |
|---|---|
| **단상 상세** | 단상 전문 / 날짜 / 왼쪽부터: 날짜 - '수정'(연보라 텍스트) - '삭제'(연보라 텍스트) - **'카드 만들기' 버튼(맨 우측)** |
| **카드 상세** | 카드 전체 모습 / 왼쪽 '삭제'(연한 테두리 버튼), 오른쪽 '수정'(채워진 보라 버튼) |
| **삭제 확인** | "이 단상을 삭제할까요?" / "이 카드를 삭제할까요?" → 취소 / 삭제 |
| **이탈 경고** | "작성 중인 내용이 있어요" → 계속 작성하기 / 나가기 |

- 모든 팝업은 **바깥 dim 배경 클릭 시 닫힘**
- **빨간색 사용 금지** — 모든 경고/삭제 버튼도 보라 팔레트로 통일

### 카드 수정 (덮어쓰기)
- 카드 상세 팝업 → '수정' → 해당 카드 내용(텍스트/출처/저자/폰트/배경)이 채워진 채로 카드 만들기 화면으로 이동
- 저장 시 **새 카드 추가가 아니라, 원래 카드 id를 찾아 덮어쓰기**
- **수정해도 최초 생성 날짜는 유지** (오늘 날짜로 바뀌면 안 됨)

### 소장하기 흐름 (단상/문장 통일)
- 오늘의 문장 '소장하기' → 바로 저장하지 않고 **'카드 만들기'의 '문장 수집하기' 모드로 이동**
- 이때 **text, source, author 모두 미리 채워져야 함** (quotes.ts의 필드명)
- 단상 목록 '카드 만들기' → 단상 내용이 채워진 채로 '단상 기록하기' 모드로 이동

### 날짜 표기 (앱 전체 통일)
- 영어 형식: **"Jul 8. 2026"**
- **시간은 표시하지 않음** (팝업, 카드 모두 날짜만)

## 6. 데이터 구조

### quotes.ts (오늘의 문장 데이터)
경로: `lib/quotes.ts`
```ts
{
  id: '15',
  text: '내 인생을 망치러 온 나의 구원자, 나의 타마코, 나의 숙희.',
```

- 매일 자정(0시) 기준으로 문장이 바뀜
- 사용자가 직접 문장을 추가/수정함
- ⚠️ 문장 안에 작은따옴표(')가 있으면 백틱(`)으로 감싸야 함

## 8. Git 백업 규칙 (사용자 요청 방식)

버전 + 날짜 태그로 관리합니다. 매 작업 단위마다 아래 5줄을 실행합니다.

```
git add -A
git commit -m "작업 내용 (YYYY-MM-DD)"
git tag -a vX.X-YYYYMMDD -m "태그 설명"
git push origin main
git push origin vX.X-YYYYMMDD
```

확인: `git log --oneline -1` / `git tag`

## 9. 🎯 다음 작업: 구글 로그인 + 기기 간 동기화

### 목적
사용자가 **폰에서 기록한 단상·카드를 PC에서도 그대로 보고 싶어 함.** 현재는 localStorage 저장이라 기기마다 데이터가 따로 놉니다.

### 방향
**Supabase**로 구글 로그인 + 데이터베이스를 함께 구축합니다. (사용자가 이전부터 Supabase 연동을 계획해왔음)

### 예상 단계
1. Supabase 계정/프로젝트 생성 — **사용자가 직접** (무료 플랜 가능)
2. Google Cloud Console에서 OAuth 클라이언트 키 발급 — **사용자가 직접**
3. Claude Code로 Supabase 연결 + 구글 로그인 구현
4. 단상·카드 저장을 localStorage → Supabase DB로 전환
5. 기존 localStorage 데이터 처리 방침 결정 (마이그레이션 vs 새 시작)

### 주의할 점
- 1·2단계는 **클로드 코드가 대신 할 수 없고 사용자가 직접** 콘솔에서 설정해야 함 → 화면 캡처를 받아 단계별로 안내 필요
- 사용자가 비개발자이므로 **API 키, 환경변수(.env), OAuth 개념**을 쉬운 말로 설명해야 함
- **환경변수(.env) 파일은 절대 git에 올라가면 안 됨** — `.gitignore` 확인 필수
- 작업 전 반드시 **git 백업(v2.1 등)** 먼저 수행
- 데이터 구조가 바뀌는 큰 작업이므로, Claude Code에 **"먼저 계획을 설명하고 확인받고 진행"** 요청 필수
